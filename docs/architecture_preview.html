4步 Pipeline 执行顺序
LLM Agent (需 API Key)
规则 Worker (无需 Key)
持久化 (JSON + SQLite)
0
Signal Collection
SignalCollector.collect(keywords) → XHS / Douyin / IG 或 Mock JSON
输出: demo/output/trend_signals.json
1
Trend Analysis
有 Key → TrendScoutAgent（Qwen3-235B / OpenRouter）调用 search_xhs / douyin / ig → save_trend_analysis
无 Key → trend_analyst.analyse() 规则排名
输出: TrendAnalysisResult { top_10, style_trends, patterns, anomalies }
2
Value Eval + Asset Gen ⚡ 并行 ThreadPoolExecutor
2a: value_evaluator.evaluate() → MetricSnapshot[] → demo/output/metric_snapshots.json
2b: asset_generator.generate() → drafts[] → demo/output/style_cards_draft.json
3
Campaign Strategy
有 Key → CampaignAgent 调用 load_trend_context → check_xhs_compliance → save_campaign_card → finalise_campaign
无 Key → campaign_strategist.strategise() 规则生成
输出: StyleCard[] (XHS / Douyin / IG 三端文案 + 定价 + 档期)
4
Summary + Memory Distill
summarizer.summarise() → SummaryReport { markdown, top_3_keywords }
memory.distill(pipeline_id) → pattern → insight 写入 SQLite
输出: demo/output/report.md
Agent 双链路 Fallback 逻辑
每个 Agent 调用前先检查 is_available()（探测 MODELSCOPE_API_KEY → OPENROUTER_API_KEY）。

✅ 成功 → Runner.run_streamed(agent, msg, max_turns=20/30) + 读磁盘结果
❌ 失败 / 异常 → _rule_based_fallback() 调同名 worker，结果结构完全一致

驱动机制： 所有状态存在 st.session_state["_chat_event_store"]，事件只追加不修改。
两阶段渲染： 先渲染 user bubble → phase=pending_start → st.rerun() → 再执行 runner，防止 UI 卡顿。
adjust_kws： 用户在 trends_review 修改关键词后，重置回 collecting 重新跑 Step 0+1。
C端 AI 试戴 — 6步调用链
1
POST /sessions（上传手部照片）
multipart 上传 → analyze_hand_image() → MediaPipe + 色彩规则 → HandProfile { hand_shape, skin_tone, undertone }
自动触发 Round 1 推荐，一并返回 session_id + 标注图
2
GET /sessions/{id}（可选，获取会话详情）
返回 session + user_image + hand_profile
3
GET /sessions/{id}/recommendations/latest（展示 Round 1）
RecommendationSnapshot — 按 hand_shape + skin_tone 规则匹配，返回排名款式列表
4
POST /sessions/{id}/events（用户行为埋点）
event_type: click / view / try_on_start → BehaviorEvent 存入 SQLite
5
POST /sessions/{id}/tryon（发起试戴）
→ InteractionService.run_tryon() → ComfyUI FLUX.2 Klein → result_image_url
GET /sessions/{id}/tryon/latest 查询结果
6
POST /sessions/{id}/recommendations/round2
需要至少 1 条行为事件 → 从历史学习视觉偏好 (色系 / 色温 / 亮度 / 饱和度) → 重新排序款式
GET /sessions/{id}/recommendations/latest?round_no=2 取结果
