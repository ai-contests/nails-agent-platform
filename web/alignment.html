🛠 问题清单 + 修复方向
① 款式名称通用（style_name = "法式甲"）

根因：Campaign Agent 上下文只有 tag 字符串（"法式"、"猫眼"），不知道具体的 display_label 和 color_tags。

修复：在 _format_trend_context() 里，对每个 StyleTrend 附加对应信号的 display_label + color_tags；LLM prompt 中要求输出 "蓝色猫眼·夏日" 而非泛名。
同时在 _build_style_card 里保留 display_label 字段。
② Tag 补全：VLM 应补 color_tags

VisionTagEnricher 已实现且 API key 已配（ModelScope Qwen2.5-VL-72B）。
问题在于 should_call_llm 判断门槛：conf ≤ 0.7 OR 空分类 ≥ 2。

当 style_tags 有 2 条时 conf = 0.71，空分类（color、material、scene）= 3 → 理论上触发。但文本 LLM 先跑，返回了空 tag（被 clean_tag 过滤），conf 没变，vision 也触发了但同样结果空。

修复：① color_tags 空时强制触发 vision（不依赖 conf 门槛）；② clean_tag 长度上限从 8 放宽到 12（"奶油裸粉"、"冷灰蓝色" 等被截断）；③ vision prompt 明确要求先写颜色。
③ 9宫格切割（已写，未 commit）

切割逻辑已在 xhs_mcp_fetcher.py 里写好（未 commit）：

  • 若帖子有合集封面（1080×1080 近正方形）且后续有单图 → 直接下单图，丢合集封面
  • 若只有合集封面 → 切割 3×3，Laplacian 选最清晰 cell
注意：XHS 搜索接口返回的封面是 640×853（ratio 0.75），永远不触发 grid9 检测。只有用户主动制作的"合集封面"（1080×1080）才触发。关键词中加入"美甲款式合集"可提升命中率。
操作：立即 commit 这段代码。
④ Top10 重复款式

10 张 style card 里：法式甲 ×4，猫眼 ×4，高级 ×1，渐变 ×1。

修复：① Campaign Agent prompt 加约束"每种 style tag 最多 2 张"；② 在 _format_trend_context 里去重，同 tag 的多条信号只保留最高热度的 1 条。
⑤ nail_styles_store 写入（memory.db 一直为空）

ingest_campaign_styles 调用 memory_store.put_style()，但 nail_styles_store 表是 0 行。

调查方向：MemoryStore() 默认用 ~/.nails_agent/memory.db，而 pipeline 里 self.memory = memory or MemoryStore()——应该是同一实例。需要确认 ingest 的 memory 参数传递是否正确。

临时做法：在 pipeline 跑完后验证 nail_styles_store 行数，如果仍为 0 则在 ingest 里加日志定位。
